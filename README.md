# 基于阿里云通义千问的多模态RAG系统

本项目是面向「全国大学生计算机设计大赛」AI方向的 **多模态知识问答系统** 原型，实现从本地知识库文件夹（如 `D:\知识库资料20`）构建检索增强生成（RAG）能力，并调用阿里云通义千问大模型进行问答与分析。

## 核心特性

- **本地知识库接入**：支持从指定文件夹递归加载文本、PDF、Word，以及图片（通过OCR提取文字）。
- **RAG 检索增强生成**：对知识库分片向量化，基于向量相似度检索相关片段，再交给通义千问生成回答。
- **多模态支持**：图片中的文字内容会被提取后一并进入知识库，实现图文混合检索。
- **阿里云大模型接入**：统一封装通义千问 DashScope 接口，便于后续更换模型或升级参数。
- **比赛导向的工程设计**：
  - 清晰的模块划分：`ingest.py`（构建索引）、`rag_service.py`（RAG 算法）、`main.py`（对话入口）、`config.py`（配置中心）。
  - 便于扩展为 Web 应用（例如基于 FastAPI + 前端页面）。

## 目录结构

- `config.py`：项目配置与常量。
- `ingest.py`：从 `D:\知识库资料20` 构建向量索引。
- `rag_service.py`：RAG 检索与调用通义千问生成回答。
- `main.py`：命令行对话入口，可作为后续 Web / 桌面应用的核心后端。
- `requirements.txt`：项目依赖列表。

## 快速开始

### 方式A：双击运行（推荐，不用手敲命令）

按顺序双击运行以下文件（都在项目根目录 `d:\rag项目`）：

- `01_一键配置密钥.cmd`：输入一次密钥，会生成本机的 `.env`
- `02_重建索引.cmd`：读取 `D:\知识库资料20` 构建索引（首次必做/知识库更新后再做）
- `03_启动网页.cmd`：启动网页服务，然后浏览器打开 `http://127.0.0.1:8000/chat-ui`
- `04_启动命令行.cmd`：启动命令行问答（可选）
- `05_打包成软件.exe生成.cmd`：一键打包生成桌面软件 `RAG启动器.exe`（可选）

> `.env` 只用于本机运行，请勿分享/上传/提交。

### 像软件一样点击使用（桌面启动器）

项目提供了一个中文图形化启动器：`launcher.py`，可一键完成“配置密钥 → 重建索引 → 启动网页 → 自动打开聊天界面”。

- 打包生成 EXE：双击 `05_打包成软件.exe生成.cmd`，完成后运行 `dist\RAG启动器.exe`
- 运行 EXE 后：
  - 若需要更换知识库目录，点“选择知识库目录”
  - 点“重建索引”生成索引文件
  - 点“启动网页”并自动打开中文聊天界面

### 方式B：命令行运行

1. **创建虚拟环境并安装依赖**

```bash
cd d:\rag项目
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

2. **配置阿里云通义千问密钥**

请在系统环境变量或 `.env` 中设置：

```bash
set DASHSCOPE_API_KEY=你的阿里云API密钥
```

> 出于安全原因，请不要把密钥写进代码仓库。

3. **构建本地知识库索引**

```bash
python ingest.py --data_dir "D:\知识库资料20"
```

4. **启动命令行问答**

```bash
python main.py
```

输入你的问题，系统会基于本地知识库 + 通义千问给出中文回答。
